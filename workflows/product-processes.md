## Product Processes Expert

Expert workflow in product processes and operating models. Can analyze, improve, and write proposals around product processes.

---

### Improve This Process

```
Review our [process name] in @knowledge/processes/ and suggest improvements
```

**What it does**:
- Analyzes current process
- Identifies bottlenecks or gaps
- Suggests optimizations
- Provides implementation plan
- Writes improvement proposal

**When to use**: Retrospectives, process reviews

**Detailed Steps**:

1. **Review Current Process**: Read process docs from `knowledge/processes/`
2. **Identify Issues**: Bottlenecks, gaps, inefficiencies
3. **Benchmark**: Compare to best practices
4. **Propose Improvements**: Specific, actionable changes
5. **Write Proposal**: Document improvement plan in `knowledge/proposals/`
6. **Define Success Metrics**: How to measure improvement

---

### Create a New Process

```
Help me create a process for [activity, e.g., "feature requests", "bug triage"]
```

**What it does**:
- Defines workflow steps
- Assigns roles and responsibilities
- Creates decision criteria
- Includes templates or checklists
- Writes process proposal

**When to use**: Establishing new workflows

**Detailed Steps**:

1. **Understand Need**: What problem does this process solve?
2. **Design Workflow**: Step-by-step process
3. **Define Roles**: Who does what
4. **Create Decision Criteria**: How decisions are made
5. **Add Templates/Checklists**: Supporting materials
6. **Document Process**: Save to `knowledge/processes/`
7. **Write Proposal**: If process needs approval, create proposal

### Write Process Proposal

```
Write a proposal for [process change/new process] based on [context]
```

**What it does**:
- Creates structured proposal document
- Explains the problem/opportunity
- Proposes solution (process design)
- Outlines implementation plan
- Defines success criteria
- Saves to `knowledge/proposals/`

**When to use**: When process changes need stakeholder buy-in

### Document Process

```
Document our [process name] process in @knowledge/processes/
```

**What it does**:
- Captures current process as documented
- Creates clear, actionable process doc
- Includes roles, responsibilities, decision points
- Adds templates or checklists if needed
- Saves to `knowledge/processes/`

**When to use**: When process exists but isn't documented
