## Write Update

### Draft an Executive Update

```
Draft an exec update on [project] for this week
```

**What it does**:
- TL;DR summary
- Progress update
- Key metrics
- Blockers and risks
- Asks and next steps

**When to use**: Weekly exec updates

**Detailed Steps**:

1. **Know Your Audience**: Exec vs. team vs. cross-functional
2. **Structure**:
   - **TL;DR**: One sentence summary
   - **Context**: What's happening and why
   - **Progress**: What's done, what's in flight
   - **Blockers**: What needs help
   - **Next Steps**: What's coming
   - **Asks**: Decisions or support needed
3. **Use Data**: Reference `knowledge/product-analytics/` for metrics
4. **Apply Their Style**: Check `knowledge/voice-samples/` for writing style if drafting communication
5. **Be Crisp**: Executives want clarity, not detail

### Write a Team Update

```
Write an update for the team on [project/sprint]
```

**What it does**:
- Celebrates wins
- Shares progress
- Calls out upcoming work
- More casual tone than exec updates

**When to use**: Sprint reviews, weekly team updates
