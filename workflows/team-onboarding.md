## Team Onboarding Expert

Expert workflow for onboarding new product managers or engineers to the team or projects, and preparing handoffs.

---

### Onboard New Team Member

```
Create an onboarding guide for a new [role: PM/engineer/designer]
```

**What it does**:
- Lists key docs to read (from `knowledge/` folders)
- Explains team processes (from `knowledge/processes/`)
- Provides context on current work (initiatives, tasks)
- Suggests first week activities
- Introduces team structure and key stakeholders

**When to use**: Before new hire starts

**Detailed Steps**:

1. **Identify Role**: PM, Engineer, Designer, etc.
2. **Gather Context**:
   - Key docs from `knowledge/company-context/`
   - Product strategy from `knowledge/product-strategy/`
   - Current initiatives from `initiatives/`
   - Team processes from `knowledge/processes/`
3. **Create Onboarding Plan**:
   - Week 1: Core context and setup
   - Week 2: Deep dive into current work
   - Week 3: Start contributing
4. **List Key Documents**: Prioritized reading list
5. **Identify Key People**: Who to meet and why
6. **Set Up Access**: Tools, systems, repositories

---

### Prepare Project Handoff

```
Create a handoff document for [project/area] to [person/team]
```

**What it does**:
- Current state summary
- Key decisions and context (from `knowledge/proposals/`)
- Open questions and blockers
- Next steps and recommendations
- Links to relevant docs and resources

**When to use**: Transitions, vacations, project transfers

**Detailed Steps**:

1. **Gather Project Context**:
   - Initiative files from `initiatives/`
   - Specs/PRDs from `knowledge/briefs-and-specs/`
   - Decisions from `knowledge/proposals/`
   - Tasks from `tasks/`
2. **Document Current State**: Where things stand now
3. **Capture Key Decisions**: Why decisions were made
4. **List Open Questions**: What needs to be resolved
5. **Define Next Steps**: What should happen next
6. **Provide Resources**: Links to all relevant docs

### Prepare Team Handoff

```
Create a handoff for my [role/responsibilities] to [person/team]
```

**What it does**:
- Summarizes current responsibilities
- Lists active initiatives and projects
- Documents key relationships and stakeholders
- Provides context on ongoing work
- Suggests transition timeline

**When to use**: Role transitions, team changes, extended leave

---

### Create Onboarding Checklist

```
Create a detailed onboarding checklist for [role] starting [date]
```

**What it does**:
- Day-by-day checklist for first 2 weeks
- Week-by-week plan for first month
- Links to required reading
- Meeting schedule with key people
- First tasks to take on

**When to use**: Structured onboarding for new team members

---

### Knowledge Transfer Session Prep

```
Help me prepare for a knowledge transfer session on [topic/project]
```

**What it does**:
- Creates session agenda
- Prepares talking points
- Identifies key documents to share
- Lists questions to cover
- Suggests follow-up materials

**When to use**: Before knowledge transfer meetings
